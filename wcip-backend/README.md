# World Cup Intelligence Platform — Backend (FastAPI)

Production-structured FastAPI service that wraps the prediction engine and
exposes match/tournament/scenario prediction, auth, saved simulations, and
admin analytics. Runs locally with **zero external services** (SQLite +
in-memory cache + eager Celery) and scales to Postgres + Redis + Celery in prod.

> All output is statistical simulation for education and analysis — not betting advice.

- 18/18 tests passing (`pytest`)

## Architecture

```
HTTP ─▶ FastAPI (app/main.py)
          │  CORS · rate-limit · JWT auth · global error handler
          ▼
        Routers (app/api/v1/*)
          │
          ├─ services/prediction.py ──▶ engine (wcip/*)   ← single bridge
          │       small runs: inline          large runs: Celery task
          ├─ repositories/repos.py  ──▶ SQLAlchemy ──▶ Postgres / SQLite
          └─ core/cache.py          ──▶ Redis (fallback: in-memory)

Celery worker (app/workers) ─▶ run_simulation, refresh_data (beat, 6h)
```

Request flow for a large simulation: `POST /simulations` persists a row,
dispatches `run_simulation.delay(id)` to Celery, returns `task_id`; the worker
runs the Monte Carlo engine and writes the result back; the client polls
`GET /simulations/{id}`.

## Local setup

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env                 # optional; defaults work as-is
uvicorn app.main:app --reload        # http://localhost:8000/docs
```

Tables auto-create and teams auto-seed on startup (idempotent). To use Alembic
instead: `alembic upgrade head`.

### Full stack with Docker

```bash
docker compose up --build            # api:8000, postgres:5432, redis:6379, worker, beat
```

## Security

JWT access/refresh (separate token types, validated by `type` claim), bcrypt
password hashing (72-byte safe), role-based admin guard, per-IP fixed-window
rate limiting, CORS allow-list, Pydantic validation on every payload, ORM-only
queries (no string SQL). Set a real `SECRET_KEY` and a Postgres `DATABASE_URL`
in production.

## Deployment

- **Render Python web service**:
  - Root Directory: `wcip-backend`
  - Build Command: `pip install -r requirements.txt`
  - Start Command: `bash scripts/start_render.sh`
  - Direct app path: `app.main:app`
  - Direct command: `uvicorn app.main:app --host 0.0.0.0 --port $PORT`
  - Required env: `DATABASE_URL`, `SECRET_KEY`, `JWT_SECRET_KEY`,
    `JWT_REFRESH_SECRET_KEY`, `ALLOWED_ORIGINS`, `ENVIRONMENT=production`
  - `DATABASE_URL` comes from Render PostgreSQL's Internal Database URL.
  - Generate secrets with
    `python -c "import secrets; print(secrets.token_urlsafe(64))"`.
  - Set `ALLOWED_ORIGINS` to
    `https://world-cup-intelligence-platform.vercel.app,http://localhost:3000`.
- **Any Docker host**: `docker compose up` or build the image and run the
  `uvicorn` and `celery worker` commands against managed Postgres/Redis.
